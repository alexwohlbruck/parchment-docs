Barrelman is the self-hosted OSM search and spatial engine that powers Parchment's place search, category lookup, and spatial queries. It runs as a separate service in the Docker Compose stack.

Source: [github.com/alexwohlbruck/barrelman](https://github.com/alexwohlbruck/barrelman)

---

## Overview

Barrelman ingests an OSM PBF extract (from Geofabrik) into PostGIS, then exposes a REST API for:

- **Nearby search** — find places within a radius, filtered by category
- **Text + semantic search** — full-text, fuzzy, abbreviation, and vector search layers
- **Spatial containment** — find parent areas (city/county/state) for a point
- **Children lookup** — list POIs inside a venue or area (e.g. shops inside a mall)
- **Place detail** — fetch a single OSM element by type + ID

Parchment's server connects to Barrelman over a shared Docker network. The `BarrelmanIntegration` handles all communication and maps responses to Parchment's unified `Place` model.

## Prerequisites

In addition to the standard Parchment prerequisites, you'll need:

- [osm2pgsql](https://osm2pgsql.org) ≥ 1.10
  ```bash
  brew install osm2pgsql    # macOS
  sudo apt install osm2pgsql  # Ubuntu/Debian
  ```
- [DuckDB CLI](https://duckdb.org) (used by some import utilities)
  ```bash
  brew install duckdb
  ```

## Setup

### 1. Clone the Barrelman repository

Clone it alongside the Parchment repo (they share a Docker network):

```bash
git clone https://github.com/alexwohlbruck/barrelman.git
cd barrelman
cp .env.example .env
```

The default `.env` works for local development. You only need to change `IMPORT_BBOX` if importing a region other than North Carolina.

### 2. Create the shared Docker network

Both Parchment and Barrelman use an external `parchment-network`. Create it once:

```bash
docker network create parchment-network
```

### 3. Start Barrelman infrastructure

```bash
cd barrelman
docker compose up -d barrelman-db ollama
```

Wait ~15 seconds for PostGIS to initialise before running the import.

### 4. Install Barrelman dependencies

```bash
bun install
```

### 5. Import OSM data

The import pipeline downloads a PBF extract, loads it into PostGIS, and builds search indexes. Choose a region to import:

```bash
# Default: North Carolina (~10-15 min)
bun run import:osm
```

For other regions, set `GEOFABRIK_URL` in `.env` before running:

```dotenv
# Germany
GEOFABRIK_URL=https://download.geofabrik.de/europe/germany-latest.osm.pbf
IMPORT_BBOX=5.9,47.3,15.0,55.1

# France
GEOFABRIK_URL=https://download.geofabrik.de/europe/france-latest.osm.pbf
IMPORT_BBOX=-5.1,41.3,9.6,51.1

# Global (no bbox filtering)
GEOFABRIK_URL=https://planet.openstreetmap.org/pbf/planet-latest.osm.pbf
IMPORT_BBOX=-180,-90,180,90
```

<Callout type="info">
The import script automatically downloads the PBF if not already present at `data/region.osm.pbf`. Subsequent runs reuse the cached file.
</Callout>

### 6. Generate embeddings (optional)

Semantic search uses Ollama vector embeddings. This step is optional — all other search layers (full-text, fuzzy, abbreviation) work without it.

```bash
# Pull the model (one-time, ~270 MB)
docker exec barrelman-ollama ollama pull nomic-embed-text

# Generate embeddings (~30-90 min on CPU for a US state)
bun run import:embed
```

Skip this if you don't need semantic/concept search (e.g. "somewhere quiet to study").

### 7. Start the Barrelman server

```bash
bun run dev
```

Barrelman runs at `http://localhost:3001`. Interactive Swagger docs at `http://localhost:3001/swagger`.

## Connecting to Parchment

Parchment's server connects to Barrelman via the `barrelman` integration. In your Parchment `.env`, configure the integration:

```dotenv
BARRELMAN_HOST=http://barrelman:3001      # Docker service name when both run in compose
BARRELMAN_API_KEY=brm_dev_changeme        # Must match Barrelman's BARRELMAN_API_KEY
```

When running the full stack with `./start.sh dev`, Parchment's `docker-compose.yml` connects to the `parchment-network` network where Barrelman is also present.

<Callout type="warn">
If Barrelman isn't running or its API key doesn't match, Parchment degrades gracefully — search still works via Pelias for addresses, but OSM POI search will return no results.
</Callout>

## Running the full stack

To run Parchment + Barrelman together:

```bash
# Terminal 1: Start Parchment
cd parchment
./start.sh dev

# Terminal 2: Start Barrelman (already imported)
cd barrelman
bun run dev
```

Or add Barrelman to your Parchment Docker Compose via `include`:

```yaml
# parchment/docker-compose.override.yml
include:
  - ../barrelman/docker-compose.yml
```

## Import pipeline details

The `bun run import:osm` command runs these steps in sequence:

| Step | Script | Description |
|------|--------|-------------|
| 1 | Download PBF | wget from Geofabrik (skipped if file exists) |
| 2 | `osm2pgsql` | Imports all OSM nodes/ways/relations via flex Lua style |
| 3 | `post-import.sql` | Extracts structured address/contact fields, builds GiST + GIN indexes |
| 4 | `generate-abbreviations.ts` | Pre-computes `name_abbrev` for autocomplete (UNCC → UNC Charlotte) |
| 5 | tsvector rebuild | Rebuilds full-text search vectors to include abbreviations |

After import, the `geo_places` table contains all importable OSM objects with:
- `centroid` — point geometry used for distance queries and indexing
- `geom` — full geometry (polygon/linestring/point) for spatial containment queries
- `tags` — raw OSM tag dictionary (JSONB)
- `categories` — array of `id-tagging-schema` category IDs
- `ts` — pre-computed `tsvector` for full-text search
- `embedding` — 768-dim float32 vector (populated by `import:embed`)

## API authentication

All Barrelman endpoints require a Bearer token:

```
Authorization: Bearer brm_dev_changeme
```

The token is set via `BARRELMAN_API_KEY` in Barrelman's `.env`. The same value must be configured in Parchment's integration settings. In development the default key is fine; change it before any public deployment.

## Troubleshooting

**`osm2pgsql: command not found`**
Install osm2pgsql: `brew install osm2pgsql` (macOS) or `sudo apt install osm2pgsql` (Ubuntu).

**Import fails with PostGIS connection error**
Ensure the `barrelman-db` container is running and fully initialised:
```bash
docker compose logs barrelman-db
docker compose ps barrelman-db   # should show "healthy"
```

**No search results in Parchment**
Check that:
1. Barrelman is running: `curl http://localhost:3001/health`
2. The API key in Parchment's integration config matches `BARRELMAN_API_KEY`
3. The import completed without errors: the `geo_places` table should have rows

**Semantic search not working**
Ollama may not be running or the model isn't pulled:
```bash
docker exec barrelman-ollama ollama list   # should show nomic-embed-text
curl http://localhost:11434/api/tags       # should return model list
```

Barrelman silently skips the semantic layer when Ollama is unavailable — other search layers continue to work.
